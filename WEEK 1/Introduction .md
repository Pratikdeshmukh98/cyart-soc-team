
Theoretical Learning:
The theoretical component establishes the conceptual and framework-level understanding required for effective security operations. In this phase, I focused on core 
security principles such as the CIA triad, risk modeling (threat–vulnerability–risk relationship), and defensive architectures including defense-in-depth and zero 
trust. I also aligned my understanding with industry standards and frameworks like NIST and MITRE ATT&CK to ensure structured and standardized knowledge.

This learning enabled me to interpret how and why security controls are implemented, how threats exploit weaknesses, and how SOC processes are designed. It provides 
the analytical foundation necessary to evaluate alerts, understand attack behavior, and apply appropriate response strategies in real-world scenarios.

Practical Learning:

The practical component focused on applying theoretical knowledge in a controlled lab and host-based environment to simulate real SOC activities. I performed hands-
on tasks such as log analysis using Windows Event Viewer, detecting brute-force attempts, parsing browser artifacts, configuring SIEM dashboards, and creating alert 
rules in tools like Elastic SIEM and Wazuh.

Through these exercises, I validated detection techniques, observed how security events are generated and correlated, and documented incidents using structured 
formats. This approach ensured that I not only understood concepts but also demonstrated the ability to implement, analyze, and respond to security events using 
actual tools and data, reflecting real-world SOC workflows.
