**Capstone Project: Comprehensive SOC Incident Response:**

1. Introduction

In this capstone project, I performed a comprehensive end-to-end simulation of a cybersecurity incident to replicate real-world Security Operations Center (SOC) 
workflows. The primary objective was to integrate theoretical knowledge and practical skills across multiple domains, including threat detection, alert triage, 
incident response, adversary emulation, post-incident analysis, automation, and executive reporting.

This project was designed to simulate the lifecycle of a sophisticated attack, beginning from initial exploitation to final reporting and continuous improvement. 
By leveraging multiple industry-standard tools, I aimed to evaluate detection capabilities, validate response mechanisms, and measure SOC performance using key 
metrics. The exercise provided a holistic understanding of how different components of a SOC environment interact to identify, contain, and analyze cyber threats 
effectively.

2. Activities Overview

I executed a full incident response lifecycle that included:

-Simulating an attack using exploitation techniques
-Detecting malicious activity through monitoring tools
-Triaging alerts and managing incidents
-Containing and mitigating the threat
-Automating response actions using SOAR principles
-Performing adversary emulation to validate defenses
-Conducting post-incident analysis and root cause identification
-Measuring SOC performance using metrics
-Preparing detailed technical and executive reports

3. Tools I Used
-Metasploit – Used to simulate exploitation of a vulnerable system
-Wazuh – Used for real-time monitoring, log analysis, and alert generation
-CrowdSec – Used for enforcing response actions such as IP blocking
-TheHive – Used for case management and incident tracking
-MITRE Caldera – Used to simulate adversary TTPs and validate detection
-Elastic Security – Used for log analysis and metrics visualization
-Google Docs – Used for documentation and reporting

4. Attack Simulation

I initiated the project by simulating an attack on a vulnerable Metasploitable2 virtual machine using Metasploit. I exploited the Samba usermap script vulnerability (exploit/multi/samba/usermap_script), which allowed unauthorized remote access. This step represented a real-world exploitation scenario targeting misconfigured or unpatched services.

The successful exploitation confirmed that the system was vulnerable and established the initial foothold for further analysis and detection.

5. Adversary Emulation

To extend the attack scenario, I used MITRE Caldera to simulate adversary behavior mapped to MITRE ATT&CK T1210 (Exploitation of Remote Services). This allowed me 
to replicate attacker techniques in a controlled environment and test SOC detection capabilities.

Wazuh successfully detected the activity, generating alerts with relevant details such as timestamp, source IP, and attack description. This validated that the 
monitoring and detection mechanisms were functioning as intended.

6. Detection and Triage

Once the attack was detected, I analyzed the alerts generated by Wazuh. The alerts indicated suspicious exploitation activity originating from a specific IP 
address. Based on severity and potential impact, I classified the alert as high priority.

I then used TheHive to create and manage the incident case. This included documenting alert details, tracking investigation progress, and organizing response 
actions. The triage process ensured that the incident was properly categorized and handled in a structured manner.

7. Response and Containment

To mitigate the threat, I implemented immediate response actions:

Isolated the compromised virtual machine to prevent lateral movement
Blocked the attacker’s IP address using CrowdSec
Verified containment by performing network tests (e.g., ping)

These actions ensured that the attack was effectively contained and prevented further exploitation.

8. SOAR Automation

To improve efficiency and consistency, I developed a SOAR playbook that automated key response actions. The playbook was designed to:

Automatically create an incident case in TheHive
Enrich alerts with contextual data
Block malicious IP addresses
Log all actions for auditing and tracking

I verified the execution of the playbook, confirming that automation reduced response time and minimized manual intervention.

9. Post-Incident Analysis

After containment, I conducted a detailed post-incident analysis to identify the root cause and contributing factors. I applied the 5 Whys technique to determine that the vulnerability existed due to outdated patches and insufficient system hardening.

Additionally, I created a Fishbone Diagram to categorize contributing factors into areas such as technology, process, and people. This provided a comprehensive understanding of the incident and highlighted areas for improvement.

10. Metrics Reporting

To evaluate SOC performance, I calculated key metrics based on the incident:

-Mean Time to Detect (MTTD): 2 hours
-Mean Time to Respond (MTTR): 4 hours
-Dwell Time: Derived from detection delay

Using Elastic Security, I created a dashboard to visualize these metrics and analyze trends. The results indicated moderate detection efficiency but highlighted the need for faster response and improved automation.

11. Reporting and Documentation

I documented the entire incident in a structured report, including:

-Executive Summary
-Detailed Incident Timeline
-Root Cause Analysis
-Metrics and Performance Evaluation
-Recommendations for improvement

I also prepared a stakeholder briefing tailored for non-technical audiences, summarizing the incident, its impact, and key improvements in a clear and concise 
manner.

12. Findings
-Successfully simulated and detected a real-world exploitation scenario
-Effective alert generation and incident triage using Wazuh and TheHive
-Successful containment through system isolation and IP blocking
-Automation significantly improved response efficiency
-Root cause identified as lack of patch management and system hardening
-Metrics revealed opportunities for improving response time

13. Outcome

This capstone project provided a complete understanding of SOC operations by integrating detection, response, automation, analysis, and reporting into a single 
workflow. It demonstrated how multiple tools and techniques work together to handle complex security incidents.

The exercise strengthened my ability to think analytically, respond to threats efficiently, and continuously improve security processes. It also reinforced the importance of a proactive and data-driven approach in modern cybersecurity operations, ensuring better preparedness against evolving threats.
